# 贡献指南

感谢你对本项目的关注！我们欢迎各种形式的贡献。

## 🤝 如何贡献

### 报告 Bug

如果你发现了 Bug，请：

1. 检查是否已有相关 Issue
2. 如果没有，创建新 Issue，包含：
   - 清晰的标题
   - 详细的问题描述
   - 重现步骤
   - 预期行为和实际行为
   - 环境信息（OS、Node.js 版本、Clawdbot 版本）
   - 相关日志（记得隐去敏感信息）

### 提出新功能

1. 先创建 Issue 讨论功能需求
2. 等待维护者反馈
3. 获得认可后再开始开发

### 改进文档

文档改进非常欢迎！包括：

- 修正错别字
- 改进表述
- 添加示例
- 翻译文档
- 添加截图

直接提交 Pull Request 即可。

### 贡献代码

1. Fork 本仓库
2. 创建功能分支：`git checkout -b feature/amazing-feature`
3. 提交更改：`git commit -m 'Add amazing feature'`
4. 推送到分支：`git push origin feature/amazing-feature`
5. 创建 Pull Request

## 📝 代码规范

### 提交信息

使用清晰的提交信息：

- `feat: 添加新功能`
- `fix: 修复 Bug`
- `docs: 更新文档`
- `style: 代码格式调整`
- `refactor: 重构代码`
- `test: 添加测试`
- `chore: 构建/工具相关`

### 文档规范

- 使用简体中文
- 使用 Markdown 格式
- 代码块注明语言
- 添加适当的标题层级

### 脚本规范

- 使用 Bash 编写
- 添加详细注释
- 处理错误情况
- 提供友好的用户提示

## 🔍 Pull Request 流程

1. **确保通过测试**
   - 手动测试所有更改
   - 确保脚本可执行
   - 验证文档格式

2. **更新相关文档**
   - 如果更改了功能，更新对应文档
   - 如果添加了配置项，更新配置示例

3. **保持整洁**
   - 一个 PR 只做一件事
   - 清理调试代码
   - 移除不必要的注释

4. **等待审核**
   - 维护者会尽快审核
   - 根据反馈进行修改
   - 保持沟通
